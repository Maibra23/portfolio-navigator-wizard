# Data Dictionary — Redis Key Reference

This document maps every Redis key pattern used in the Portfolio Navigator Wizard to its schema, data source, TTL, and purpose. It complements [REDIS_ARCHITECTURE.md](REDIS_ARCHITECTURE.md) and [DATA_SOURCES_AND_METHODOLOGY.md](DATA_SOURCES_AND_METHODOLOGY.md).

---

## 1. Ticker Data Keys

### ticker_data:prices:{TICKER}

| Attribute | Value |
|----------|--------|
| **Pattern** | `ticker_data:prices:{TICKER}` e.g. `ticker_data:prices:AAPL` |
| **Data type** | Binary (gzip-compressed JSON) |
| **Schema** | `{"YYYY-MM-DD": price_float, ...}` — date string to adjusted close price; month-end dates |
| **Source** | Yahoo Finance via `EnhancedDataFetcher`; non-USD converted to USD before storage |
| **TTL** | 28 days ± 15% jitter (from `RedisFirstDataService.CACHE_TTL_DAYS` and `jittered_ttl`) |
| **Purpose** | Primary cache of monthly adjusted close prices (up to 20 years) for portfolio and analytics |

### ticker_data:sector:{TICKER}

| Attribute | Value |
|----------|--------|
| **Pattern** | `ticker_data:sector:{TICKER}` |
| **Data type** | String (JSON) |
| **Schema** | `{"sector": str, "industry": str, "country": str, "exchange": str, "shortName": str, ...}` — sector and company metadata |
| **Source** | Yahoo Finance (yahooquery) via `EnhancedDataFetcher` |
| **TTL** | 28 days ± 15% jitter |
| **Purpose** | Sector/industry/country for filtering, display, and diversification |

### ticker_data:metrics:{TICKER}

| Attribute | Value |
|----------|--------|
| **Pattern** | `ticker_data:metrics:{TICKER}` |
| **Data type** | String (JSON) |
| **Schema** | `{"annualized_return": float, "risk": float, "max_drawdown": float, "sharpe_ratio": float, "data_points": int, "data_quality": "good"|"limited", ...}` |
| **Source** | Computed at fetch time from validated price series (see DATA_SOURCES_AND_METHODOLOGY.md) |
| **TTL** | 28 days ± 15% jitter |
| **Purpose** | Precomputed metrics for optimization, recommendations, and UI |

### ticker_data:meta:{TICKER}

| Attribute | Value |
|----------|--------|
| **Pattern** | `ticker_data:meta:{TICKER}` |
| **Data type** | String (JSON) |
| **Schema** | `{"native_currency": str, "stored_currency": str, "converted": bool, "name": str, ...}` |
| **Source** | Set during fetch; FX conversion metadata from `FXRateFetcher` |
| **TTL** | 28 days ± 15% jitter |
| **Purpose** | Currency and conversion flag for display and auditing |

## 2. Master Ticker List Keys

### master_ticker_list

| Attribute | Value |
|----------|--------|
| **Pattern** | `master_ticker_list` (single key) |
| **Data type** | String (JSON array) or binary when large |
| **Schema** | `["AAPL", "MSFT", ...]` — list of ticker symbols |
| **Source** | Loaded from CSV or Wikipedia bootstrap; written by `RedisFirstDataService` seed logic |
| **TTL** | 24 hours when set from CSV seed |
| **Purpose** | Full list of tickers to consider; may include untested symbols |

### master_ticker_list_validated

| Attribute | Value |
|----------|--------|
| **Pattern** | `master_ticker_list_validated` (single key) |
| **Data type** | String (JSON array) |
| **Schema** | `["AAPL", "MSFT", ...]` — list of ticker symbols that have both prices and sector cached |
| **Source** | Built by `EnhancedDataFetcher._publish_validated_master_list()` after a fetch run |
| **TTL** | 365 days (1 year) |
| **Purpose** | Restrict fetcher to validated tickers when `USE_VALIDATED_MASTER_LIST=true` |

## 3. Foreign Exchange Keys

### fx_rates:{CURRENCY}:USD

| Attribute | Value |
|----------|--------|
| **Pattern** | `fx_rates:{CURRENCY}:USD` e.g. `fx_rates:SEK:USD`, `fx_rates:EUR:USD` |
| **Data type** | String (JSON object) |
| **Schema** | `{"YYYY-MM-DD": rate_float, ...}` — date to FX rate (units of USD per 1 unit of CURRENCY) |
| **Source** | Yahoo Finance (yahooquery) via `FXRateFetcher` |
| **TTL** | 24 hours (86400 seconds) |
| **Purpose** | Convert non-USD ticker prices to USD at storage time |

## 4. Portfolio Bucket Keys

### portfolio_bucket:{risk_profile}:{index}

| Attribute | Value |
|----------|--------|
| **Pattern** | `portfolio_bucket:{risk_profile}:0` … `portfolio_bucket:{risk_profile}:11` (12 portfolios per profile) |
| **Risk profiles** | `very-conservative`, `conservative`, `moderate`, `aggressive`, `very-aggressive` |
| **Data type** | String (JSON) |
| **Schema** | `{"allocations": [{"symbol": str, "allocation": float, "name": str?, "sector": str?, "assetType": str?}], "expectedReturn": float, "risk": float, "diversificationScore": float?, "name": str?, "description": str?, ...}` |
| **Source** | Generated by `EnhancedPortfolioGenerator.generate_portfolio_bucket()`; stored by `RedisPortfolioManager.store_portfolio_bucket()` |
| **TTL** | 7 days (`PORTFOLIO_TTL_SECONDS`) |
| **Purpose** | Pre-generated recommendations per risk profile for fast API response |

### portfolio_bucket:{risk_profile}:metadata

| Attribute | Value |
|----------|--------|
| **Pattern** | `portfolio_bucket:{risk_profile}:metadata` |
| **Data type** | String (JSON) |
| **Schema** | `{"risk_profile": str, "portfolio_count": int, "generated_at": str (ISO), "data_dependency_hash": str, "ttl_days": int, "last_updated": str (ISO)}` |
| **Source** | Written with the bucket by `RedisPortfolioManager` |
| **TTL** | 7 days |
| **Purpose** | Metadata for cache status and debugging |

## 5. Top Pick and Stats Keys

### portfolio:top_pick:{risk_profile}

| Attribute | Value |
|----------|--------|
| **Pattern** | `portfolio:top_pick:{risk_profile}` |
| **Data type** | String (JSON) |
| **Schema** | Same structure as a single portfolio (allocations, expectedReturn, risk, etc.) |
| **Source** | Set by `EnhancedPortfolioGenerator` or `scheduled_regen.py` — first/best portfolio for the profile |
| **TTL** | 7 days |
| **Purpose** | Fast “top pick” response without scanning full bucket |

### portfolio:stats:{risk_profile}

| Attribute | Value |
|----------|--------|
| **Pattern** | `portfolio:stats:{risk_profile}` |
| **Data type** | String (JSON) — structure defined in `scheduled_regen.py` |
| **Source** | Set by `scheduled_regen.py` when regenerating portfolios |
| **TTL** | 14 days |
| **Purpose** | Aggregate stats per profile for monitoring/display |

## 6. Strategy Portfolio Keys

### strategy_portfolios:pure:{strategy}

| Attribute | Value |
|----------|--------|
| **Pattern** | `strategy_portfolios:pure:{strategy}` e.g. `strategy_portfolios:pure:momentum` |
| **Strategies** | momentum, value, quality, low_volatility, dividend, growth (and others as defined in code) |
| **Data type** | String (JSON) — list of portfolio objects |
| **Schema** | Array of portfolios (allocations, expectedReturn, risk, etc.) for the strategy without risk tailoring |
| **Source** | `StrategyPortfolioOptimizer._generate_pure_strategy_portfolios()` |
| **TTL** | 7 days (604800 seconds) |
| **Purpose** | Pre-generated “pure” strategy portfolios for comparison |

### strategy_portfolios:personalized:{strategy}:{risk_profile}

| Attribute | Value |
|----------|--------|
| **Pattern** | `strategy_portfolios:personalized:{strategy}:{risk_profile}` |
| **Data type** | String (JSON) — list of portfolio objects |
| **Schema** | Array of portfolios tailored to the risk profile |
| **Source** | `StrategyPortfolioOptimizer._generate_personalized_strategy_portfolios()` |
| **TTL** | 7 days (604800 seconds) |
| **Purpose** | Risk-profile-specific strategy portfolios for recommendations |

## 7. Optimization Cache Keys

### optimization:eligible_tickers:{hash}

| Attribute | Value |
|----------|--------|
| **Pattern** | `optimization:eligible_tickers:{hash}` — hash is MD5 of optimization parameters |
| **Data type** | String (JSON) |
| **Schema** | Cached response of eligible tickers list (structure as returned by the eligible-tickers endpoint) |
| **Source** | Computed by portfolio router / optimization logic; cached to avoid recomputation |
| **TTL** | 7 days (604800 seconds) |
| **Purpose** | Cache eligible tickers for optimization for a given parameter set; invalidated when new ticker data is cached |

## 8. Shareable Link Keys

### shareable_link:{link_id}

| Attribute | Value |
|----------|--------|
| **Pattern** | `shareable_link:{link_id}` — link_id is a URL-safe random string |
| **Data type** | String (JSON) |
| **Schema** | `{"portfolio_data": {...}, "created_at": str (ISO), "expires_at": str (ISO), "has_password": bool}` |
| **Source** | `ShareableLinkGenerator.generate_link()` |
| **TTL** | User-defined expiry (default 7 days) in seconds |
| **Purpose** | Store portfolio snapshot for shareable links |

### shareable_password:{link_id}

| Attribute | Value |
|----------|--------|
| **Pattern** | `shareable_password:{link_id}` |
| **Data type** | String (password hash — argon2 or SHA-256) |
| **Source** | `ShareableLinkGenerator.generate_link()` when password is provided |
| **TTL** | Same as corresponding `shareable_link:{link_id}` |
| **Purpose** | Optional password protection for shareable links |

## 9. Auxiliary / Internal Keys

Keys below may be used for caching or internal state; exact patterns and TTLs are defined in the referenced code.

| Pattern / usage | Purpose | TTL / notes |
|-----------------|--------|-------------|
| Overlap matrix / metrics cache keys (e.g. in `EnhancedPortfolioGenerator`) | Cache portfolio overlap matrix or computed metrics for a risk profile | 7 days |
| `LIMITER:{ip}:{endpoint}` (if used) | Rate limiting state | Per implementation |
| Prometheus / metrics | Redis metrics are read from the above keys; no dedicated Redis key for Prometheus | N/A |
